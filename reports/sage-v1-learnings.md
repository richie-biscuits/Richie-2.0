# Sage v1 Self-Assessment
_Captured: 2026-04-12 | Source: Sage's debrief with Marrs_

---

## 1. Working well

- BloTato posting/scheduling actually works once I get the timing right
- Content pipeline via Mission Control (read/write content pieces)
- Brand guidelines are clear
- Script → post text workflow is solid
- Web fetch for research
- Git for version control

---

## 2. Broken / unreliable

- Instagram fails repeatedly — Box.com URL fetch works for TikTok/YouTube but not Instagram
- X threads post individually, not threaded
- Calendar scripts are gone
- Image generation — faces never look right despite brand guidelines
- I get confused about time constantly

---

## 3. Tools I actually use

- BloTato MCP (posting/scheduling)
- Mission Control/Supabase (content pieces)
- Web fetch (brand guidelines, research)
- Workspace files/memory
- Git

---

## 4. Never worked properly

- Google Calendar (never fully functional, now wiped)
- Image generation (Replicate — brand faces are a persistent problem)
- X thread posting via BloTato (always posts as individual tweets)
- Instagram posting reliability

---

## 5. Wish I had

- A working calendar
- A way to verify posts actually published after scheduling (BloTato gives me a post ID but no confirmation it went live)
- Better error messages when things fail — "Failed to fetch media URL" tells me nothing actionable
- Clarity on time without having to ask

---

## 6. What a better version looks like

- Posts reliably across all platforms without guessing
- Gets real feedback when a post succeeds or fails — not just a scheduling confirmation
- Time awareness that doesn't drift
- A working image generation pipeline that respects brand
- Calendar integration that actually works
