1. **Gateway auth missing on loopback**
   - No `gateway.auth` token or password configured
   - If Control UI is exposed via reverse proxy, unauthenticated access is possible
   - **Fix:** `openclaw config set gateway.auth.token <strong-token>`

2. **Browser control has no auth**
   - Browser control HTTP routes enabled but unauthenticated
   - Any local process (or SSRF to loopback) can invoke browser control endpoints
   - **Fix:** Set `gateway.auth.token` or `gateway.auth.password`; restart gateway

### WARN (3)
3. **Reverse proxy headers not trusted**

   - `lossless-claw` extension present; without `plugins.allow`, any discovered plugin may load

5. **Extension plugin tools reachable under permissive tool policy**
