Platform Profiles

OpenClaw

Our core orchestration layer
Orchestration

Multi-agent coordination platform running locally or cloud. Our primary build environment — where we install cognition, connect skills, and coordinate teams. Runs on Mac Mini, cloud VM, or dedicated hardware. Hit a major release on April 25 (2026.4.25) with significant TTS, plugin registry, and telemetry upgrades.

PricingCloud hosting from ~$17/mo on AWS Lightsail (basic VM). No per-seat or per-agent licensing. Self-hosted free.
Key strengthFull control, multi-agent teams, skill ecosystem, local execution, massive community
Key limitationRequires setup; not a managed SaaS; deployment UX still maturing
Best forComplex multi-agent workflows, privacy-sensitive work, custom builds, long-running autonomous agents

Polynize role: Our primary build environment. Where we install the cognition layer and coordinate agent teams. The 2026.4.25 release solidifies enterprise readiness with OTEL, improved browser automation, and plugin cold registry.

Source: github.com/openclaw/openclaw/releases

Perplexity Agent API

Multi-provider agent API with web search
Agentic API

Perplexity's Agent API is a multi-provider, interoperable API specification for building LLM applications. Access models from OpenAI, Anthropic, Google, xAI and more through one unified interface with integrated real-time web search, tool configuration, reasoning control, and token budgets. Transparent pricing — no markup over direct provider pricing. Supports structured output via JSON schema, source filtering by domain/recency, and preset configurations for specific use cases.

PricingUsage-based per token. Transparent — see exact token costs per request. No markup over provider pricing. API key via console.perplexity.ai.
Key strengthMulti-provider from one API; integrated web search; transparent pricing; JSON schema output; source filtering; preset modes (pro-search); reasoning control
Key limitationNo local execution; consumed as external service; less customisation than building directly with provider APIs
Best forResearch-heavy workflows, multi-model routing, web-grounded responses, structured data extraction at scale
OpenClaw via APIYes — Perplexity API can be called from OpenClaw agents as an external reasoning/research engine. Connect via HTTP.

Polynize role: Useful as a research module within OpenClaw workflows — especially for multi-source web research and structured data extraction. Transparent pricing makes it cost-predictable for client billing.

Source: docs.perplexity.ai | Agent API Quickstart

Claude Managed Agents

Anthropic's managed agent infrastructure — public beta
Managed Agents

Claude Managed Agents entered public beta on April 9, 2026. Pre-built, configurable agent harness running in Anthropic's managed infrastructure. Claude gains access to bash, file operations, web search & fetch, and MCP servers within secure cloud containers. Supports long-running execution (minutes/hours), stateful sessions, prompt caching, and compaction. Multi-agent and outcomes features in research preview.

PricingAPI usage-based. Claude Sonnet: ~$3/input + $15/output per million tokens. Opus: ~$15/$75. Managed Agents usage-based (beta pricing TBC for GA). Rate-limited: 300 create/min, 600 read/min per org.
Key strengthBest-in-class reasoning + autonomous execution. Secure containers. Long context (200K). Built-in tool harness. Prompt caching. SSE streaming. MCP support.
Key limitationStill in beta. Multi-agent/outcomes in research preview only. Closed model. Walled garden — no local execution. At scale Opus gets expensive.
Best forLong-running autonomous tasks, document processing, code generation, strategic analysis, enterprise deployments wanting managed infrastructure

Polynize role: The most significant competitive development this month. If Managed Agents goes GA with competitive pricing, it becomes a credible alternative for clients who want "agents as a service" without infrastructure management. Our cognition layer remains the differentiator — anyone can use Managed Agents, but we make them think like domain experts.

Source: platform.claude.com/docs/en/managed-agents/overview | claude.com/blog/claude-managed-agents

n8n

Workflow automation backbone
Automation

Open-source workflow automation. Connects APIs, databases, and AI services into triggered pipelines. Self-hosted or cloud. 500+ integrations. Supports human-in-the-loop, AI guardrails, SSO/SAML, audit logs, version control, and RBAC. Adding more AI-specific nodes and Claude integration.

PricingFree self-hosted. Cloud: from ~€20/mo. Enterprise: custom. On-prem option, SSO SAML, LDAP, encrypted secret stores.
Key strength500+ integrations, workflow triggers, enterprise governance (RBAC, audit logs, SSO), human-in-the-loop, AI guardrails, git-based version control
Key limitationNot a thinking agent — follows fixed pipelines; needs Claude/OpenClaw for complex decisions; AI node capabilities still maturing
Best forCRM syncs, notification pipelines, data transformation, scheduled report generation, operational automation

Polynize role: The automation backbone under OpenClaw agents. Agents make decisions, n8n executes the fixed workflows those decisions trigger. Strong enterprise governance features make it viable for regulated clients.

Source: n8n.io

NVIDIA NemoClaw

NVIDIA's secure OpenClaw variant — early preview
Enterprise Security

NVIDIA NemoClaw is an open-source stack that adds privacy and security controls to OpenClaw. Announced March 16, 2026 at GTC. Uses NVIDIA OpenShell to enforce policy-based privacy and security guardrails on agent behaviour and data handling. Adds sandboxed execution, network policies, privacy-preserving model routing, and audit logging. Can run local models like NVIDIA Nemotron for enhanced privacy. Single-command install: curl -fsSL https://www.nvidia.com/nemoclaw.sh | bash. Early preview.

PricingOpen-source, free to install. Local inference uses your own hardware. Cloud model routing available via privacy router.
Key strengthSecurity/privacy guardrails on OpenClaw; local Nemotron model inference; NVIDIA ecosystem integration; single-command deploy; audit logging; network policies
Key limitationEarly preview stage; requires NVIDIA hardware/software stack for full capabilities; less community momentum than vanilla OpenClaw; limited documentation
Best forEnterprise clients requiring strict data privacy; regulated industries (healthcare, finance, government); clients who want OpenClaw but with security guarantees

Polynize role: Relevant for enterprise deals where privacy and data governance are hard requirements. Builds on OpenClaw — we can deploy NemoClaw stacks with our cognition layer on top. For regulated industry clients, this could be the entry point that gets us past security reviews.

Source: nvidia.com/en-au/ai/nemoclaw | docs.nvidia.com/nemoclaw

Base44 Superagents

No-code agent builder — consumer/SMB focus
No-Code Agents

Base44 Superagents are no-code personal AI agents that live in WhatsApp, Telegram, Slack, iMessage, or browser. No Docker, servers, SSH, or config files required. Takes real actions — sending messages, following up with leads, pulling reports, reacting to events. Remembers context across sessions. Has built-in connectors for Calendly, Contentful, Hugging Face, Supabase, Google Meet, Google Tasks, and more. Active development — GPT-5.5 just added (April 24).

PricingFree tier available. Pro plans from ~$30/mo. Enterprise: custom (Elite/Enterprise with data residency options).
Key strengthZero setup friction; works where clients already work (WhatsApp, Slack, iMessage); proactive automation; built-in connectors; remembers/evolves; GPT-5.5 and Opus 4.7 support
Key limitationNo local execution; walled garden; limited customisation beyond connectors; not designed for complex multi-agent orchestration; users hitting limits and looking for alternatives
Best forQuick personal productivity agents; SMB automation; clients who want agents without technical setup; simple lead follow-up and notification workflows

Polynize role: Good for entry-level client demos and simple personal automation use cases. Their rapid feature velocity (iMessage, Telegram, skills, security scans all added in April) shows they're investing. But for anything requiring deep customisation, complex workflows, or enterprise scale — we migrate to OpenClaw + our cognition layer.

Source: base44.com | docs.base44.com/changelog

The Bottom Line

The architecture that wins: For any complex client problem, the answer usually involves OpenClaw as the orchestration layer, Claude as the intelligence, and our cognition layer installed on top — telling the whole system how to think like an expert in that client's domain.

Biggest shift this week: OpenClaw's v2026.4.25 release is significant — OpenTelemetry, cold plugin registry, and PWA/Web Push move the platform toward enterprise readiness. This is exactly what we need as our core build platform. Combined with Claude Managed Agents (beta) and NVIDIA NemoClaw (preview), the infrastructure layer is maturing fast. Our differentiator — the cognition layer — becomes more valuable as the platforms commoditise.
