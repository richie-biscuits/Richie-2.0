Platform Profiles

OpenClaw

Our core orchestration layer
Orchestration

Multi-agent coordination platform running locally or cloud. Our primary build environment — where we install cognition, connect skills, and coordinate teams. Runs on Mac Mini, cloud VM, or dedicated hardware. The operating system for personal AI.

PricingCloud hosting from ~$17/mo on AWS Lightsail (basic VM). No per-seat or per-agent licensing.
Key strengthFull control, multi-agent teams, skill ecosystem (ClawHub), local execution, open-source
Key limitationRequires setup; not a managed SaaS; cloud deployment UX still a pain point
Best forComplex multi-agent workflows, privacy-sensitive work, custom builds

Polynize role: Our primary build environment. Where we install the cognition layer and coordinate agent teams.

Source: openclaw.ai

Perplexity Computer

Multimodal AI agent with web + API capabilities
Agentic AI

Perplexity Computer is a category-defining multimodal AI agent that reasons across text, images, audio, and video. Capabilities include autonomous web browsing and interaction, document understanding and extraction, code generation and execution, real-time data synthesis, and multi-step task completion. The API exposes agent endpoints with support for multiple frontier models (Sonnet, GPT, Gemini). Can be integrated into OpenClaw via HTTP/MCP.

PricingAPI: usage-based per token. Pro subscription ~$20/mo for consumer access. Enterprise pricing TBC.
Key strengthMultimodal reasoning; autonomous web browsing; real-time synthesis; source filtering; multi-model API support; code execution
Key limitationClosed platform; limited customization beyond API; consumed as external service
Best forDeep market research, document analysis, competitor research, investment due diligence, multimodal content synthesis
OpenClaw via APIYes — Perplexity API callable from OpenClaw agents as external reasoning engine via HTTP/MCP.

Polynize role: Integrate via API into OpenClaw as a research module. We install cognition on top so it knows how to research like an expert in the client's domain — not just consume data.

Source: perplexity.ai/products/computer | docs.perplexity.ai

Claude Managed Agents

Anthropic's managed agent harness
Autonomous Agents

Claude Managed Agents provides a pre-built, configurable agent harness running in Anthropic's managed cloud infrastructure. Best for long-running tasks and asynchronous work. No need to build your own agent loop, sandbox, or tool execution layer — it handles prompt caching, compaction, and performance optimizations out of the box. Four core concepts: Agent (model + tools + skills), Environment (container config), Session (running instance), Events (message exchange). Steer or interrupt sessions mid-execution.

PricingClaude 3.5 Sonnet: ~$3/input + ~$15/output million tokens. Opus 4: ~$15/$75. Managed Agents: usage-based (alpha — pricing TBC on GA).
Key strengthBest-in-class reasoning; long context (200K); managed infrastructure; safety alignment built in; session persistence; prompt caching
Key limitationManaged Agents still alpha; closed model; walled gardens emerging (Anthropic restricting OpenClaw connectors); expensive at Opus scale
Best forComplex reasoning, document analysis, autonomous task execution, coding, strategic thinking, long-running async work

Polynize role: Primary intelligence model for most builds. Our cognition layer tells Claude how to think about the specific domain — not just what to do. As Managed Agents matures, this becomes an increasingly powerful build target.

Source: platform.claude.com/docs/en/managed-agents/overview

n8n

AI workflow automation platform
Automation

Self-described "AI workflow automation platform." Open-source workflow automation connecting APIs, databases, and AI services into triggered pipelines. Self-hosted or cloud. Strong enterprise positioning with security and governance features. Now claims 500+ integrations (up from 300+).

PricingFree self-hosted. Cloud: from ~€20/mo. Enterprise: custom pricing.
Key strengthMassive integration library (500+ apps), workflow triggers, reliability, AI governance features (human-in-the-loop, guardrails, evaluations)
Key limitationNot a thinking agent — follows fixed pipelines; needs OpenClaw or similar for complex decisions
Best forCRM syncs, notification pipelines, data transformation, scheduled report generation, AI governance-heavy workflows

Polynize role: The automation backbone under OpenClaw agents. Agents make decisions, n8n executes the fixed workflows those decisions trigger. Notable: they've added "AI governance" language — human-in-the-loop, guardrails, evaluations — which signals a shift toward enterprise AI workflow trust and safety.

Source: n8n.io

Nemoclaw

NVIDIA's secure OpenClaw variant
Orchestration

NVIDIA Nemoclaw is an open-source stack that adds privacy and security controls to OpenClaw. One-command install: curl -fsSL https://www.nvidia.com/nemoclaw.sh | bash. Uses NVIDIA OpenShell to enforce policy-based guardrails on agent behaviour and data handling. Runs high-performance open models like NVIDIA Nemotron locally for enhanced privacy and cost efficiency. Early preview stage.

PricingOpen-source, free to install. Local inference uses your own hardware. Cloud model routing available via privacy router.
Key strengthSecurity/privacy guardrails on OpenClaw; NVIDIA OpenShell enforcement; local Nemotron inference option; single-command deploy; NVIDIA ecosystem
Key limitationEarly preview stage; requires NVIDIA hardware/software stack; less community momentum than vanilla OpenClaw
Best forEnterprise clients requiring strict data privacy; regulated industries; clients who want OpenClaw with security guarantees

Polynize role: Relevant for enterprise deals where privacy and data governance are hard requirements. Builds on OpenClaw — we can deploy Nemoclaw stacks with our cognition layer on top.

Source: nvidia.com/en-au/ai/nemoclaw

Base44 Superagents

No-code agent builder — consumer/SMB focus
No-Code Agents

Base44 Superagents are no-code personal AI agents that live in WhatsApp, Telegram, Slack, or browser. No Docker, servers, SSH, or config files required. They take real actions — sending messages, following up with leads, pulling reports, reacting to events — and remember context across sessions. Base44 also positions as a full app-building platform (not just agents), which creates strategic ambiguity about where Superagents sit in their product hierarchy.

PricingFree tier available. Pro plans from ~$30/mo. Enterprise plans available.
Key strengthZero setup friction; works where clients already work (WhatsApp, Slack); proactive automation; built-in connectors
Key limitationNo local execution; walled garden; limited customisation beyond connectors; product focus split between app-builder and agents
Best forQuick personal productivity agents; SMB automation; clients who want agents without technical setup

Polynize role: Good for entry-level client demos and simple personal automation use cases. For anything requiring deep customisation, complex workflows, or enterprise scale — we migrate to OpenClaw + our cognition layer.

Source: base44.com | base44.com/superagents

The Bottom Line

The architecture that wins: For any complex client problem, the answer usually involves OpenClaw as the orchestration layer, Claude as the intelligence, and our cognition layer installed on top — telling the whole system how to think like an expert in that client's domain.

When to go deeper: Add Perplexity Computer for research-heavy problems. Add n8n for operational automation that doesn't need judgment. The stack is modular — we pick what fits. For enterprise with governance requirements, consider Nemoclaw.

n8n's shift matters: They went from "workflow automation" to "AI workflow automation platform" with explicit AI governance features. This is the market moving toward Polynize's territory — and it validates our positioning. We're ahead of the curve.

Our differentiator: Anyone can spin up a Claude agent. We install the cognition layer that makes it think like a domain expert. That's the Polynize difference — and it's platform-agnostic.
